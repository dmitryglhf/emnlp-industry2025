# Summary

## TL;DR  
The paper introduces a two‑stage pipeline for sentence casing in e‑commerce ads. First, an LLM‑based Chain‑of‑Actions (CoA) agent explicitly identifies brand names and enforces correct casing via a brand hub library. Second, the agent’s knowledge is distilled into a lightweight Flan‑T5‑Small model that uses an in‑context constrained decoding algorithm to avoid hallucinations. The resulting system matches GPT‑4o performance while reducing latency by an order of magnitude, making it suitable for real‑time ad compliance.

## Problem & Motivation  
E‑commerce ad platforms such as Amazon, Target and Walmart enforce sentence‑casing policies to maintain readability and brand consistency. Existing NER‑based transformer models treat each character as a classification problem, lacking sentence‑level context and struggling with unseen or ad‑specific terms like brand names. Commercial LLMs such as ChatGPT generalize better but are expensive and have high latency, and generative models can hallucinate new entities. Thus, an automated, scalable, and cost‑effective sentence‑casing solution is required for real‑world ad compliance.

## Approach  
**Core idea:** a two‑stage pipeline.  
1. **LLM CoA Agent** – given ad text, the agent first identifies potential special terms (brand names). If a brand is detected, it queries a **Brand Hub Library** to retrieve the correct spelling and capitalization, then incorporates the retrieved brand name into the prompt to generate sentence‑cased text. This decomposes the task into sub‑tasks: identify, verify, generate.  
2. **Distilled generative model** – a Flan‑T5‑Small encoder‑decoder is pre‑trained on public news data (6 M pairs, 50 % positive/negative). It is then fine‑tuned on 22 k ad‑specific pairs (positive/negative balanced). During inference, an **in‑context constrained decoding** algorithm is used: a prefix trie over the token vocabulary is constructed; at each step only tokens whose lower‑case matches the remaining input substring are allowed, preventing hallucinations and ensuring only casing changes. This preserves the LLM agent’s knowledge while reducing latency and cost.

## Experimental Setup  
| Dataset | Type | Size | Augmented | Pos/Neg |
|---------|------|------|-----------|---------|
| Public News | Train | 6 M | Yes | 50 % / 50 % |
| | Evaluation | 4 k | Yes | 50 % / 50 % |
| Real Ads | Train | 22 k | Yes | 50 % / 50 % |
| | Evaluation | 1 k | No | 15 % / 85 % |

Real‑ads data were collected from 3 500 historical ads, yielding 7 700 text samples (headlines, sub‑headlines, CTAs).  
**Baselines**:  
- NER: XLM‑Roberta fine‑tuned for Truecasing.  
- GEC: GRMR‑V3‑G4B, GRMR‑V3‑Q1.7B (fine‑tuned from Gemma and Qwen).  
- LLMs: GPT‑4o, GPT‑4, Llama‑2‑13b‑chat, Llama‑2‑70b‑chat.  
**Metrics**: False‑Transformation Rate (FTR), Precision, Recall, F1.  
**Deployment context**: offline evaluation on datasets; latency and cost for the distilled model are reported in Table 5 (values not shown).
