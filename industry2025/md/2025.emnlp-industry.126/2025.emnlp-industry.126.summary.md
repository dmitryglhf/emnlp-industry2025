# Summary

## TL;DR  
The paper proposes a data‑efficient framework for Automatic Prompt Optimization (APO) tailored to memory‑enhanced conversational agents.  By holistically optimizing the prompts for memory writing, memory reading, and response generation, and by employing an LLM‑based evaluator that also attributes errors to specific modules, the authors achieve efficient prompt refinement with minimal human annotation.  The method is evaluated on two health‑related datasets using Gemini‑2.5‑pro Coma and Llama3‑70b, showing improved consistency and personalization scores.

## Problem & Motivation  
Memory‑enhanced conversational agents store, retrieve, and utilize historical interactions to provide personalized responses.  Optimizing prompts for such agents is difficult because:  
1. They involve multiple interconnected LLM calls (write, read, generate).  
2. A sub‑optimal final response can stem from any of the three modules.  
3. Feedback for APO must identify which module caused the failure, yet human evaluation is costly, especially in specialized domains like healthcare.  
The paper addresses these challenges by proposing a holistic APO framework that integrates error attribution and data‑efficient human‑in‑the‑loop sampling.

## Approach  
1. **Holistic APO** – The prompt set \(Z=\{z_{\text{write}}, z_{\text{read}}, z_{\text{gen}}\}\) is updated jointly via  
   \[
   Z^{\text{new}} = f_{\text{APO}}\!\bigl(\{(x_{\text{user}},x_{\text{history}},m_{\text{write}},m_{\text{read}},y,s,a)_i\}, Z^{\text{old}}\bigr)
   \]  
   (Eq. 5).  
2. **LLM‑based Evaluator with Error Attribution** – For each response \(y\) the evaluator returns a score \(s\) and an attribution \(a\) indicating whether the error originates from memory writing, memory reading, or generation:  
   \[
   a,s = f_{\text{eval}}\!\bigl(y, x_{\text{user}},x_{\text{history}},m_{\text{write}},m_{\text{read}},Z^{\text{old}}\bigr)
   \]  
   (Eq. 6).  
3. **Data‑Efficient Active Sampling** – Human annotation is limited to a subset \(D_{\text{opt}}\) of size \(k\).  The subset is selected by maximizing uncertainty \(u_x\) (variance of LLM scores) while ensuring representativeness via KL divergence over topic and error‑attribution distributions:  
   \[
   \max_{\mathbf{w}}\frac{1}{|D_{\text{unlabeled}}|}\sum_{x}w_x u_x
   -\lambda_1 KL(P||Q)-\lambda_2\|\mathbf{w}\|_1
   \]  
   (Eq. 8).  
4. **Subset Selection for APO** – For APO itself, a second convex objective selects low‑uncertainty, representative samples for prompt refinement:  
   \[
   \min_{\mathbf{w}}\frac{1}{|D_{\text{scored}}|}\sum_{x}w_x u_x
   +\lambda_1 KL(P||Q)+\lambda_2\|\mathbf{w}\|_1
   \]  
   (Eq. 9).  
5. **Implementation** – The optimization is solved with `cvxpy`; the final prompts are updated using the chosen samples and the LLM’s suggestions.

## Experimental Setup  
- **Datasets**  
  1. **HealthBench** (Arora et al., 2025) – public health‑conversation logs, augmented with simulated user messages that require memory.  
  2. **Proprietary HealthCoaching** – 500 cases collected during a one‑month user trial; each case contains user health data (step count, heart rate, skin temperature, sleep duration), user message, response, and full conversation history.  
- **Models** – Prompt optimization is evaluated with **Gemini‑2.5‑pro Coma** and **Llama3‑70b** (Grattafiori et al., 2024).  
- **Evaluation Metrics** – Consistency and Personalization, scored on a 0–2 scale by a panel of 8 professional health coaches.  
- **Human‑in‑the‑loop** – Human experts annotate a subset of samples selected by the convex objective; LLM‑based evaluator provides initial scores and error attribution.  
- **Procedure** –  
  1. Start with initial prompts for all agents.  
  2. Generate and evaluate responses for the entire training set.  
  3. Apply the active sampling strategy to select samples for human verification.  
  4. Select representative samples for APO.  
  5. Run APO to update prompts.  
  6. Evaluate initial vs. optimized prompts on a held‑out test set.
