# Summary

## TL;DR  
LEAF is a three‑stage framework that boosts factual accuracy of large language models (LLMs) in medical question answering. It introduces RAFE, a robust fact‑checking system built on Qwen2‑72B‑Instruct and a domain‑specific retrieval corpus; Fact‑Check‑then‑RAG, a retrieval‑augmented generation pipeline that uses fact‑check results to guide retrieval; and a self‑training loop (SFT + SimPO) that fine‑tunes the LLM on fact‑checked outputs. In a real‑world healthcare deployment with 208 proprietary documents, LEAF raised factuality scores from 0.438 to 0.803 (an 83 % improvement).

## Problem & Motivation  
Medical QA demands high factual accuracy because incorrect answers can have serious consequences. Existing fact‑checking methods (Factcheck‑GPT, Factscore, SAFE) rely on proprietary models (e.g., ChatGPT‑3.5) and Google Search, limiting their use in sensitive domains and on private datasets. Retrieval‑augmented generation (RAG) can introduce irrelevant context, further degrading accuracy. The paper targets these gaps by providing an open‑source, retrieval‑aware fact‑checking system that can be deployed on proprietary corpora.

## Approach  
1. **RAFE** – a Retrieval‑Augmented Factuality Evaluator that splits a response into statements, rewrites them to be self‑contained, generates context‑aware retrieval queries, retrieves evidence (three per statement) from a corpus covering Wikipedia, PubMed, StatPearls, and Medical Textbooks, and rates each statement as *Supported* or *Not Supported*. The final factuality score is the proportion of supported statements.  
2. **Fact‑Check‑then‑RAG** – after an LLM generates a response, RAFE evaluates it. If any statement is *Not Supported*, the system retrieves documents from MedCORP using ColBERT and injects them into a new prompt, forcing the LLM to regenerate a corrected answer. This avoids parameter updates while leveraging the same LLM.  
3. **Learning from Fact‑Check** – two self‑training modes:  
   * **Supervised Fine‑Tuning (SFT)** – fine‑tune the LLM on responses that achieve a factuality score of 1.0.  
   * **Preference‑Based Learning (SimPO)** – rank generated responses with RAFE, select the top‑scoring as *chosen* and the lowest as *rejected*, and train the LLM to prefer factually correct outputs.

## Experimental Setup  
| Component | Model | Baselines | Datasets | Retrieval Corpus | Retrieval Model | Evaluation Metrics |
|-----------|-------|-----------|----------|------------------|------------------|--------------------|
| RAFE | Qwen2‑72B‑Instruct | Factcheck‑GPT | MedQA, MMLU‑M, PubMedQA, BioASQ, MedMCQA | Wikipedia, PubMed, StatPearls, Medical Textbooks | – | Precision, Recall, F1 |
| Fact‑Check‑then‑RAG | LLaMA 3 70B Instruct | CoT, RAG | Same five datasets | MedCORP | ColBERT | Accuracy (percentage) |
| Self‑Training (SFT/SimPO) | LLaMA 3 8B Instruct | Factcheck‑GPT, RAFE | Same five datasets | – | – | Accuracy (percentage) |

Inference costs were measured on MedQA: RAG required 1 call and 467.0 tokens on average, whereas FC‑RAG required 3 calls and 1050.8 tokens. Parallel processing reduced verification time from 20 s to 3–4 s per response.
