# Summary

## TL;DR  
SMART is a hybrid Task‑Oriented Dialogue (TOD) framework that fuses traditional pipeline modules with modern agentic approaches. It introduces a simplified dialogue state, proactive clarification, and a unified NLG that eliminates response redundancy. Across troubleshooting and medical domains, SMART outperforms the AutoTOD baseline on factuality, relevance, and redundancy while keeping latency low (ModernBERT 110 ms vs. claude‑3‑haiku 2.25 s). The system scales to new languages (Spanish, Arabic) and, when deployed in an e‑commerce store, reduced product return rates, demonstrating tangible industry impact.

## Problem & Motivation  
Existing TOD systems struggle with unstructured information, multilingual support, and proactive engagement. Pipeline approaches suffer from error propagation and limited scalability, while causal and agentic approaches lack robust clarification workflows. The paper targets these gaps by proposing a scalable, multilingual TOD system that can handle ambiguous user inputs and avoid duplicate responses—critical for real‑world applications such as customer service, healthcare, and hotel booking.

## Approach  
SMART fuses pipeline and agentic paradigms (Figure 1). Core components:

| Component | Key Idea | Implementation |
|-----------|----------|----------------|
| **Unstructured Knowledge Base** | Build from domain‑specific content using AutoChunker (Jain et al., 2025b) to chunk while preserving context; store chunks in a vector‑DB with embeddings from *cohere.embed‑multilingual‑v3*. | |
| **Intent Detection** | Fine‑tuned *ModernBERT* (Warner et al., 2024) on domain conversations; achieves Precision 0.891, Recall 0.881, F1 0.884, latency 110 ms. | |
| **Query Reformulation & Retriever** | LLM‑based reformulator (prompt 2) generates a refined query; retrieved top‑k chunks via cosine similarity. | |
| **Action Detection** | LLM (prompt 3) decides if clarification is needed, generating question and options. | |
| **Unified NLG** | LLM (prompt 4) generates final response; for solution‑oriented queries it uses retrieved chunks, otherwise relies on intent. Implements solution de‑duplication by interrupting after each solution and continuing from the same LLM instance until a limit is reached. | |

Design choices (e.g., fine‑tuned NLU, simplified DST, proactive clarification) directly address the limitations highlighted in Table 1.

## Experimental Setup  

- **Datasets**  
  - *Troubleshooting*: knowledge base built from historical customer‑reported issues of an e‑commerce store + user guides/manuals; a sampled subset due to proprietary constraints.  
  - *Medical*: symptoms from *Kaggle* (2016) and treatments from dataset in *Shah et al.* (2021b).  
  - Embedding model: *cohere.embed‑multilingual‑v3*.

- **Evaluation**: AutoEval‑ToD framework (Jain et al., 2025a) simulates diverse user interactions; metrics evaluated with *claude‑3‑sonnet* (Anthropic, 2023).

- **Baselines**: AutoTOD (Xu et al., 2024) using *claude‑3‑haiku* (Anthropic, 2023) and agentic Retrieval‑Augmented Generation (RAG). Both systems share the same knowledge base.

- **Metrics**: Intent detection precision/recall/F1/latency (Table 2); query reformulation scores (Table 3); retriever HitRate@k (Table 4); response quality metrics (Table 5).

## Results  

- **Intent Detection**: ModernBERT outperforms claude‑3‑haiku and claude‑3‑sonnet (Table 2), with 20× faster inference.

- **Query Reformulation**: Human and LLM judges score 0.872/0.928 (Correctness), 0.844/0.908 (Retention), 0.816/0.912 (Relevance) (Table 3).

- **Retriever**: HitRate@1 rises from 0.35 (Turn 1) to 0.45 (Turn 3); HitRate@5 from 0.56 to 0.66 (Table 4).

- **Action Detection**: Clarification Accuracy 0.81, Question Correctness 0.92, Option Relevancy 0.89 (human evaluation).

- **Unified NLG**: Compared to AutoTOD, SMART achieves higher factuality (0.744 vs. 0.670), lower redundancy (0.049 vs. 0.151), higher relevance (0.963 vs. 0.897), and higher safety compliance (0.984 vs. 0.975) (Table 5).
