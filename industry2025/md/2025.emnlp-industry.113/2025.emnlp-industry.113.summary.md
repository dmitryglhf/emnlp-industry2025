# Summary

## TL;DR
CLIMB is a fully automated, bottom‑up taxonomy builder that first distills core occupations from raw job postings via global semantic clustering, then iteratively constructs a coherent hierarchy using a reflection‑based multi‑agent framework (Generator + Evaluator). On three real‑world datasets (Palestine, Botswana, USA) CLIMB produces taxonomies that are larger, more coherent, and better aligned with the data than the state‑of‑the‑art baseline TnT and the expert‑curated ESCO, achieving higher strict agreement (≈ 0.73 on Palestine) and coverage (≈ 98 %) while remaining fully data‑driven.

## Problem & Motivation
The paper tackles the need for **region‑specific, updatable occupation taxonomies** that can be used for job recommendation, labor‑market intelligence, and policy analysis. Existing solutions are either:
* **Top‑down** – expand a pre‑existing seed, requiring expert curation and limiting adaptability.
* **Bottom‑up** – build from scratch but struggle to (i) distill a globally consistent set of core concepts from noisy corpora and (ii) construct a deep, logically coherent hierarchy.

The authors argue that a fully automated, data‑driven approach is essential for dynamic labor markets.

## Approach
CLIMB follows a **two‑stage pipeline**:

1. **Leaf‑node generation**  
   * **Job posting distillation & embedding** – an LLM labels chunks as relevant, a lightweight classifier extracts them, and the resulting “distilled” descriptions are embedded.  
   * **Semantic clustering** – a custom similarity metric is learned by training an XGBoost classifier on pairs labeled by an LLM acting as an HR expert. The classifier’s similarity scores feed an Affinity Propagation algorithm to form occupation clusters.  
   * **Leaf‑node abstraction** – an LLM abstracts each cluster into a canonical title and description; ambiguous or duplicate titles are removed and semantically equivalent nodes are merged.

2. **Hierarchical construction**  
   * A **reflection‑based multi‑agent framework** builds the tree level‑by‑level.  
   * **Generator** (LLM) proposes parent concepts and child‑parent mappings.  
   * **Evaluator** scrutinises the output for logical consistency (missing children, duplicate assignments, hallucinations).  
   * The cycle repeats until the new parent set is below a threshold.

Key design choices: global clustering to avoid local‑view redundancy, reflection to prevent error propagation, and LLMs only for high‑level reasoning while lightweight models handle bulk filtering.

## Experimental Setup
| Dataset | Size (jobs) | Train/Test split |
|---------|--------------|------------------|
| Palestine | 2 701 (English + Arabic) | 9:1 |
| Botswana | 4 854 (English) | 9:1 |
| USA | 11 285 (English) | 9:1 |

**Baselines**  
* **TnT** – state‑of‑the‑art fully automated bottom‑up method.  
* **TnT‑H** – a recursive, hierarchical variant of TnT (evaluated only on Palestine).  
* **ESCO** – expert‑curated, region‑agnostic taxonomy built on ISCO‑08 (two‑level version used for evaluation).

**Evaluation**  
* Three LLM annotators (Gemini 2.5 Flash, O4‑Mini, DeepSeek R1) label test‑set jobs using each taxonomy.  
* Metrics:  
  * **Strict Agreement** – Fleiss’ kappa requiring identical labels from all annotators.  
  * **Hierarchical Agreement** – kappa allowing shared parent at a given level.  
  * **Coverage Rate** – % of jobs assigned a specific label.  
  * **Label Utilization Rate** – % of taxonomy labels used at least once.
