# Summary

## TL;DR
L4 introduces *online dynamic mutual distillation* for lifelong language learning, enabling two student models to learn from each other without domain‑specific teachers.  Experiments on MultiWOZ, CNN/DailyMail, WikiSQL, and five NLG domains show that L4 consistently outperforms LAMOL and L2KD* on dsEM, ROUGE, IfEM, and Forgetting metrics, while reducing catastrophic forgetting.

## Problem & Motivation
The paper tackles catastrophic forgetting in continual NLP systems.  Existing distillation methods rely on pre‑trained, domain‑specific teachers, which is impractical for on‑device or online‑serving models that must adapt to evolving user queries.  The authors argue that a peer‑to‑peer distillation scheme would remove this dependency and better suit real‑world, dynamic NLP applications such as adaptive chat‑bots and personalized language systems.

## Approach
L4 builds on LAMOL’s generative replay and replaces teacher‑student distillation with *mutual learning*:

1. Two identical student models, \(\theta_{S_1}\) and \(\theta_{S_2}\), are trained in parallel.  
2. Each model receives a combined supervised loss (QA + LM) on the current task and a distillation loss (KL divergence) on the other model’s predictions.  
3. For past tasks, supervised loss is applied only to pseudo‑samples generated by the models, while distillation loss is applied to the current task samples.  
4. The total loss for each model is  
   \[
   L_{\theta_{S_i}} = L_{\text{past}}(\theta_{S_i}) + \lambda \cdot L_{\text{cur}}(\theta_{S_i}),
   \]
   where \(\lambda\) balances supervised and distillation terms.  
5. The method uses the same pre‑trained small GPT‑2 model for all lifelong learning experiments.

## Experimental Setup
- **Datasets**: MultiWOZ (WOZ), CNN/DailyMail (CNN), WikiSQL (SQL); plus five NLG domains: E2E‑NLG, REST, HOT, TV, LAPTOP.  
- **Baselines**: Finetune, LAMOL, L2KD* (uses pretrained teacher models).  
- **Metrics**: Dialog State Exact Match (dsEM), ROUGE, Interaction‑Level Exact Match (IfEM), Forgetting Metric (FM).  
- **Training**: Sequential training with generative replay; two models trained jointly per epoch (Algorithm 1 in Appendix A).  
- **Implementation**: Same small GPT‑2 pre‑trained model across all LLL models.
