# Summary

## TL;DR
FLOW‑BENCH is a high‑quality, 101‑sample benchmark of natural‑language (NL) instructions paired with business‑process‑automation (BPA) artifacts (BPMN). FLOW‑GEN is an LLM‑driven pipeline that first translates NL into a concise Python intermediate representation (IR) and then deterministically converts the IR into BPMN or DMN. The approach achieves state‑of‑the‑art IR accuracy on FLOW‑BENCH, with the best activity retriever (Activities_Search) and demonstration retriever (CE_Retriever) yielding an exact‑match score of 0.7624.

## Problem & Motivation
Enterprise workflows are typically built with drag‑and‑drop tools that still require software‑engineer intervention for complex integrations and custom logic. Existing low‑code platforms lack robust natural‑language interfaces, and LLMs struggle to generate BPMN directly due to its verbosity and lack of training data. There is also no well‑established benchmark for evaluating NL‑driven workflow generation. FLOW‑BENCH and FLOW‑GEN aim to fill these gaps by providing a realistic dataset and a practical LLM pipeline.

## Approach
1. **Python IR** – NL is first mapped to a constrained subset of Python syntax (assignments, `if`, `for`, `while`, function calls). This keeps the output short (≈25× shorter than BPMN) and leverages LLMs’ strong code‑generation capabilities.
2. **Deterministic Converters** –  
   * `PY2BPMN` converts the generated Python IR to BPMN XML.  
   * `BPMN2PY` converts existing BPMN to Python for incremental edits.  
   * `DIFF2BPMN` computes the diff between input and generated Python and applies the corresponding updates to the original BPMN.
3. **Retrieval Modules** –  
   * **Activity retrievers**: `ED_Retriever` (edit distance), `Embeddings_Retriever` (Bi‑Encoder with *all‑MiniLM‑L6‑v2*), `Activities_Search` (fine‑tuned Bi‑Encoder).  
   * **Demonstration retrievers**: `TopKRetriever` (Bi‑Encoder) and `CE_Retriever` (cross‑encoder *stsb‑distilroberta‑base*).  
   * The retrieved activities (top‑k) and demos are injected into the LLM prompt.
4. **LLM Generation** – The prompt (NL utterance + activity descriptions + demos) is fed to a pre‑trained LLM (Granite‑20b‑code‑instruct‑v2 or codellama‑34b‑instruct‑hf) to produce the Python IR.

## Experimental Setup
| Component | Details |
|-----------|----------|
| Dataset | FLOW‑BENCH: 101 incremental build tests (linear + conditional). |
| API catalog | 546 endpoints from GitHub, Jira, Slack, Salesforce, etc. |
| Evaluation | Accuracy of generated Python IR (Activities Recall, Exact Match, Hallucination Rate). |
| Models | Granite‑20b‑code‑instruct‑v2, codellama‑34b‑instruct‑hf. |
| Retrieval | Top‑k activity retrievers (10, 50, 100) and demo retrievers (2, 3, 5, 7). |
| Metrics | Exact‑match score, recall, hallucination rate. |
| Deployment | Offline evaluation on the 101 tests; no online A/B‑test reported. |

## Results
| Retriever | TopK | Activities Recall | Exact Match | Hallucination Rate |
|-----------|------|-------------------|-------------|--------------------|
| ED_Retriever | 10 | 0.7327 | 0.495 | 0.0469 |
| Embeddings_Retriever | 10 | 0.9307 | 0.6733 | 0.0207 |
| Activities_Search | 10 | 0.9703 | 0.6931 | 0.0069 |
| **Activities_Search (TopK = 50)** | 50 | **0.9926** | **0.7723** | **0.0102** |

| Model | DemoSelector | TopK | Exact Match |
|-------|--------------|------|-------------|
| Granite‑20b‑code‑instruct‑v2 | TopKRetriever | 2 | 0.5842 |
| | CE_Retriever | 5 | **0.7624** |
| codellama‑34b‑instruct‑hf | CE_Retriever | 5 | **0.7624** |

*Best performance*: Activities_Search with TopK = 50 and CE_Retriever with 5 demos, achieving an exact‑match of 0.7624. Hallucination rates remain below 1 % for the best configurations.
