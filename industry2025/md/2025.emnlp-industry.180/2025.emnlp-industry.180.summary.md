# Summary

## TL;DR  
RedOne is a domain‑specific large language model (LLM) tailored for social networking services (SNS).  Built on the Qwen2.5 checkpoint, it undergoes a three‑stage post‑training pipeline—continue pretraining (CPT) on 100 B tokens, supervised fine‑tuning (SFT) on 8 core SNS tasks, and preference optimization (PO) with DPO—yielding a 7 B model that outperforms leading proprietary baselines on SNS‑Bench and SNS‑TransBench, improves online harmful‑content exposure by 11.23 % and post‑view search click‑through by 14.95 %.

## Problem & Motivation  
SNS data is informal, context‑sensitive, and emotionally charged, making content‑management and interaction‑quality tasks difficult for generic NLP systems.  Existing LLM adaptations focus on isolated tasks, suffer diminishing returns with data scaling, and fail to flexibly adapt to diverse real‑world contexts.  A domain‑specific LLM that can generalize across many SNS tasks is therefore needed.

## Approach  
RedOne follows a **three‑stage training strategy**:

1. **Continue Pretraining (CPT)**  
   * Data: 100 B tokens from (i) high‑quality open corpora (Qiu et al., 2024; Weber et al., 2024; Penedo et al., 2024) and (ii) SNS‑specific domain data collected from SNS platforms and the open web, enriched with user interaction signals.  
   * Filtering: rule‑based filtering + small‑LM filtering (Wang et al., 2025) + RegMix (Liu et al., 2024a) → 20 B high‑quality tokens.  
   * Model: Qwen2.5 checkpoint → domain‑aware CPT.

2. **Supervised Fine‑Tuning (SFT)**  
   * Tasks: 8 core SNS capabilities (Table 1) – content understanding, information extraction, semantic matching, user behavior modeling, dialogue, and translation.  
   * Data: SNS‑specific data + general instruction data (Li et al., 2025; Zhou et al., 2023a; Zhao et al., 2024; Guha et al., 2025; Ye et al., 2025).  
   * Two‑step mixed fine‑tuning: step 1 – mix SNS + general data; step 2 – higher proportion of SNS data.

3. **Preference Optimization (PO)**  
   * Preference data: constructed per task type (subjective vs. objective) using human annotations, judge‑model consistency, and implicit label structure.  
   * Fine‑tuning: DPO (Rafailov et al., 2023) with SFT loss coefficient 0.3.

## Experimental Setup  
* **Model sizes**: 7 B and 32 B (learning rates: 5 × 10⁻⁶ / 3 × 10⁻⁶ for 7 B, 3 × 10⁻⁶ for 32 B).  
* **Benchmarks**:  
  * General: MMLU, CMMLU, CEVAL, GPQA‑Diamond, News‑Bench, MMLU‑Pro, BBH, GaokaoBench, AIME2025, GSM8K, MATH500, HumanEval, MBPP, LiveCodeBench, WMT‑22/23/24, Flores, IFEval, HaluEval, CompassBench.  
  * SNS‑specific: SNS‑Bench (Guo et al.) and SNS‑TransBench (Guo et al., 2025).  
* **Baselines**:  
  * Llama‑3.1‑8B (Grattafiori et al., 2024)  
  * Mistral‑8B (Mistral‑AI, 2024)  
  * InternLM3‑8B (Cai et al., 2024)  
  * GLM‑4‑9B‑0414 (GLM et al., 2024)  
  * Qwen2.5‑7B (Qwen et al., 2025b)  
  * RedOne‑7B (ours)  

* **Evaluation metrics**: average scores per benchmark, BLEU/chrF++ for translation, task‑specific metrics (e.g., accuracy for taxonomy, hashtag Q, query‑corr M, MRC NE, NED, etc.).  
* **Deployment context**: online testing on harmful‑content detection and post‑view search.
