# Summary

## TL;DR  
The paper introduces **Isomorphic Math Problem Generation (IMPG)**, a task that produces structurally consistent variants of existing math problems for educational deployment.  It proposes **Computational Blueprints for Isomorphic Twins (CBIT)**, a meta‑generation framework that outputs a generator program rather than individual problems, guaranteeing mathematical correctness and structural consistency while dramatically reducing token cost.  In a real‑world deployment, CBIT‑generated items achieved a **17.8 % lower error rate** than expert‑authored problems, generating **186,870 interactions** for **6,732 learners** on a commercial platform.

## Problem & Motivation  
- Repetitive practice with similar problems is essential for conceptual learning, but producing such items at scale is costly because each item must be pedagogically validated.  
- Existing automatic generation methods focus on data augmentation and evaluate similarity with BLEU/ROUGE/METEOR or downstream solving benchmarks (e.g., GSM8k, MATH), offering no guarantee of mathematical correctness or structural consistency.  
- No prior work explicitly targets educational deployment; human expert verification remains a hidden cost.  
- The authors therefore define **IMPG** to generate *isomorphic* variants that inherit the pedagogical validity of source items by varying only numerical values while preserving structure.

## Approach  
1. **Task Definition** – A problem is a tuple \(P=(Q,S,A)\).  IMPG requires generating \(K\) isomorphic problems \(T=\{P_1,\dots,P_K\}\) that satisfy:  
   - *Structural Equivalence*: same number of expressions and natural‑language elements.  
   - *Conceptual Correctness*: identical set of quantitative relationships \(\mathcal{R}\).  
2. **Dataset & Verification** – The authors release the **IMPG** benchmark (https://github.com/eric-jhoon/IMPG) containing **195** expert‑authored problems across **grade 4–12** and **67** curriculum units.  Each problem is annotated with **2,410** symbolic relationships (Table 1).  An automatic verification toolkit symbolically checks generated items.  
3. **Baseline – AIC‑Batch** – Re‑implement the AIC framework (Li et al. 2024a) using a *Batch* prompting strategy: the LLM generates an abstraction, a verifier program, and the full problem text.  
4. **Blueprint for Isomorphic Twin (BIT)** – The LLM generates only an abstraction; a deterministic template reconstructs the full text, tightening structural consistency.  
5. **Computational Blueprint for Isomorphic Twin (CBIT)** – The LLM writes a *generator program* that, given a random seed, outputs an abstraction and the full problem.  This meta‑generation removes the need for repeated LLM calls, guarantees correctness via deterministic execution, and reduces token cost.

## Experimental Setup  
- **Dataset**: 195 original problems, 2,410 symbolic relationships (Easy 605, Medium 832, High 973).  
- **Baselines**: AIC‑Batch (original AIC framework re‑implemented).  
- **Metrics**:  
  - *Mean Per‑Problem Success Rate (MPSR)*: ratio of successful generations per problem.  
  - *Tokens Per Success (TPS)* and *Cost Per Success (CPS)* (using per‑token costs \(c_{pr},c_{cp}\)).  
- **Deployment Context**: Offline generation followed by online deployment on a commercial education platform, serving **6,732** learners and generating **186,870** interactions.

## Results  
- **CBIT** outperforms AIC‑Batch in both generation accuracy (higher MPSR) and cost‑effectiveness (lower TPS/CPS).  
- **Error Rate**: CBIT‑generated problems exhibit a **17.8 % lower error rate** than expert‑authored items.  
- **Scale**: 186,870 interactions across 6,732 learners.  
- No explicit numeric MPSR or token counts are reported, but the qualitative improvements are emphasized.

## Industry Impact  
- Deployed on a commercial education platform at scale.  
- Achieved a **17.8 % reduction in error rate** relative to expert‑authored problems.  
- Generated **186,870** learning interactions for **6,732** learners, demonstrating practical viability and business value in large‑scale personalized math education.

Not reported.
