# Summary

## TL;DR
ARIA (Adaptive Reflective Interactive Agent) is a large‑language‑model framework that learns continuously at test time by self‑assessing uncertainty, soliciting targeted human guidance, and updating a timestamped knowledge repository.  On TikTok Pay’s customer‑due‑diligence name‑screening task, ARIA outperforms static, fine‑tuned, and other self‑learning baselines while respecting a fixed query budget.  The system has been deployed in production, serving over 150 million monthly active users.

## Problem & Motivation
Large language models (LLMs) excel when the knowledge distribution is static, but they struggle in dynamic domains where rules and domain facts change frequently—e.g., regulatory compliance or user risk screening.  Existing agents lack the ability to adapt once deployed, often requiring costly offline re‑training.  The paper focuses on the **customer due diligence (CDD)** name‑screening task on TikTok Pay, where an agent must decide whether a user matches a watch‑list hit.  The data stream is highly imbalanced (11 846 cases, 156 positives) and evolves over time, making static models unreliable and non‑compliant.

## Approach
ARIA is built around two core modules:

| Module | Purpose | Key Design |
|--------|---------|------------|
| **Intelligent Guidance Solicitation (IGS)** | Self‑assess confidence and decide whether to query a human. | Structured self‑dialogue with a fixed set of \(N_{\text{RQ}}\) reflective questions; confidence categories \(\{\text{High},\text{Moderate},\text{Low}\); query budget \(B\). |
| **Human‑Guided Knowledge Adaptation (HGKA)** | Incorporate human feedback into a timestamped knowledge repository (KR). | Knowledge items \(k=(\text{kid},K,ts_{\text{added}},ts_{\text{validated}},S,M_{\text{meta}})\); conflict detection via semantic similarity \(\tau_{\text{sim}}\); active clarification queries when ambiguity remains. |

The agent’s policy \(\pi\) combines retrieval from KR with the base LLM \(M_{\text{LLM}}\).  Retrieval uses a composite score:

\[
\text{Score}(k)=W_S(k.S)\times S_T(k,t_{\text{current}})\times S_R(k,x_i)
\]

where \(W_S\) weights validity, \(S_T\) is a recency decay, and \(S_R\) measures semantic relevance.

The learning update is:

\[
\Theta_{i+1}=f(\Theta_i,x_i,\hat{y}_i,q_i,h_i)
\]

with \(q_i\) the query (if any) and \(h_i\) the human response.

The system is implemented with the code available at **[https://github.com/yf-he/aria](https://github.com/yf-he/aria)**.

## Experimental Setup
| Component | Details |
|-----------|----------|
| **Dataset** | TikTok Pay CDD name‑screening: 11 846 real‑world cases, 156 Match (positive) cases. |
| **Baselines** | Static Agent (Qwen2.5‑7B, GPT‑4o); Offline Fine‑tuning; RAG Agent (static KB); Active Learning (Random Querying); Active Learning (Simple Uncertainty Sampling); Self‑Refine (Madaan et al., 2023); Reflexion (Shinn et al., 2023); Multi‑Agent Debate (Du et al., 2023). |
| **Budget** | Human query budget \(B\) varied across experiments (B = 50, 100, 500, 1000). |
| **Metrics** | Sensitivity (Recall) and Specificity (1‑False‑Positive Rate). |
| **Evaluation** | Offline evaluation on the chronological stream; no online A/B‑test numbers reported. |

## Results
- **Adaptability**: ARIA’s self‑assessment reliably triggers human queries only when confidence is Moderate or Low, respecting the budget.
- **Knowledge Integration**: The timestamped KR resolves conflicts and supersedes outdated rules, enabling the agent to keep up‑to‑date with evolving compliance rules.
- **Performance Gains**: Compared to static, fine‑tuned, and other self‑learning baselines, ARIA achieves higher sensitivity and specificity across all budgets (exact numbers not reported in the excerpt).
- **Deployment**: ARIA has been deployed on TikTok Pay’s production system, serving over 150 million monthly active users.

## Industry Impact
- **Deployment Context**: TikTok Pay’s CDD name‑screening pipeline.
- **Scale**: Over 150 million monthly active users.
- **Business Outcome**: Improved compliance and risk screening accuracy, reducing false positives and missed matches, thereby lowering regulatory risk and operational costs.
