# Summary

## TL;DR  
ECom‑Bench is the first open‑source benchmark that evaluates multimodal LLM agents for e‑commerce customer support. It combines persona‑driven user simulation, realistic task instances derived from real dialogues, and a comprehensive tool set (21 categories plus a multimodal image tool) under the Model‑Context Protocol (MCP). In extensive experiments, GPT‑4o achieves the best pass‑^3 score of **16.98 %** on 53 tasks (18 multimodal), yet still far below human performance, highlighting the difficulty of real‑world e‑commerce scenarios.

## Problem & Motivation  
E‑commerce platforms require scalable, context‑aware customer service. Existing rule‑based systems handle only single‑turn interactions, lack tool calling, and cannot process multimodal inputs. Prior evaluation frameworks either omit e‑commerce‑specific tasks (ToolBench), support only single‑turn dialogues (Web‑Shop), or ignore multimodal evaluation (τ‑Bench). Consequently, there is no comprehensive benchmark to assess LLM agents in realistic, multimodal e‑commerce support, which is essential for industry deployment.

## Approach  
ECom‑Bench’s core pipeline consists of:  
1. **User Simulation** – LLMs generate user personas (emotion, patience, style) from real e‑commerce dialogues; human reviewers ensure authenticity.  
2. **Database Construction** – JSON files encode product, logistics, order, invoice data; LLMs synthesize realistic records, then experts validate them.  
3. **Tool Definition** – 21 tool categories (product inquiry, order modification, returns, etc.) plus a multimodal image‑recognition tool, all implemented under MCP.  
4. **Task Instances** – 53 manually verified tasks (18 multimodal) extracted from authentic conversations, standardized for LLM training/evaluation.  
5. **Evaluation** – Three dimensions: database state, tool invocation, and keyword presence; pass‑^k metric measures robustness across k identical trials.

## Experimental Setup  
- **Models**: GPT‑4o, DeepSeek‑V3, Doubao‑1.5‑Pro‑32k, Qwen‑Max, Moonshot‑V1‑32k, Qwen‑VL‑Max, Doubao‑Pro‑Vision*.  
- **Agent Framework**: LangGraph + ReAct, limited to 20 turns, 600 s per task, temperature = 0.3.  
- **Dataset**: 53 tasks (18 multimodal).  
- **Metrics**: pass‑^1, pass‑^2, pass‑^3 (Table 2).  
- **Consistency Test**: 8 identical trials per task (pass‑^8).  
- **Error Analysis**: 159 trajectories (53 tasks × 3 trials) for Doubao‑1.5‑Pro‑32k.  
- **Ablation**: 8 persona combinations (emotion, patience, style) on 20 tasks, measured by pass‑^3.

## Results  
| Model | Pass^1 | Pass^2 | Pass^3 |
|-------|--------|--------|--------|
| GPT‑4o | 44.03 | 26.42 | **16.98** |
| Doubao‑1.5‑Pro‑32k | 38.99 | 23.90 | 16.98 |
| DeepSeek‑V3 | 36.48 | 21.38 | 15.09 |
| Qwen‑Max | 15.09 | 8.81 | 5.66 |
| Moonshot‑V1‑32k | 14.47 | 7.55 | 5.66 |
| Doubao‑Pro‑Vision* | 20.75 | 13.21 | 11.32 |
| Qwen‑VL‑Max* | 8.18 | 4.4 | 3.77 |

- **Consistency**: GPT‑4o’s pass‑^8 drops to **7.55 %**, indicating limited robustness across repeated trials.  
- **Error Types**: Wrong Argument, Wrong Decision, Partially Resolved (Figure 5).  
- **Dimensional Evaluation**: All agents show higher failure in the Action dimension; GPT‑4o has the lowest error rates across Action, Search, and Output (Figure 6).  
- **Persona Ablation**: Emotional state, patience, and communication style significantly affect pass‑^3 (Table 3).

## Limitations  
1. **Scope** – Only home appliances and furniture vertical; cross‑category generalization remains untested.  
2. **Data Authenticity** – Sensitive commercial data forced use of LLM‑generated synthetic data, potentially less authentic.  
3. **Scale** – 53 task instances (18 multimodal) are limited by manual creation effort.  
4. **User Simulation** – Persona dimensions rely on domain expertise without extensive theoretical validation; simulation depends on LLM‑generated behavior, which may not fully capture real customer patterns.
