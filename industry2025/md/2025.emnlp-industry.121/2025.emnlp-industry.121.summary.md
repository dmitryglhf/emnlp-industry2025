# Summary

## TL;DR  
STREAQ introduces a two‑tier selective routing framework that routes contact‑center QA queries between a cost‑efficient in‑house model and high‑capability large LMs. The system cuts daily inference cost by 48 % (from \$34,162 to \$17,842) while retaining 88.9 % of the full‑model AoI precision, enabling enterprise‑scale automated QA that balances performance and economics.

## Problem & Motivation  
Contact centers generate millions of conversations that must be evaluated for compliance and service quality. Manual QA cannot scale, and fully automated QA with large LMs (e.g., GPT‑4, Claude‑4‑Sonnet) is prohibitively expensive. Small LMs are cheap but suffer from low AoI precision, producing many false positives that erode agent trust and waste QA resources. The paper highlights the need for a cost‑effective yet reliable QA pipeline that preserves high AoI precision.

## Approach  
STREAQ’s core idea is a two‑tier routing framework:

1. **Small model (Mₛ)** – the in‑house fine‑tuned model (cost‑efficient, \$3,070 per 1.5 M conversations).  
2. **Large model (Mₗ)** – one of six large LMs: *nova‑pro*, *nova‑premier*, *o3‑mini*, *gpt‑4o*, *claude‑3.5‑haiku*, *claude‑4‑sonnet*.  
3. **Router (R)** – a RoBERTa‑based binary classifier that receives the concatenation of the original instance (question, plan, conversation) and Mₛ’s output. It decides whether to accept Mₛ’s prediction (R = 0) or to forward the instance to Mₗ (R = 1).  
4. **Inference Paradigm** – If routed to Mₗ, the large model receives Mₛ’s reasoning as context, enabling it to refine the answer.  
5. **Cost Model** – Total cost is the sum of Mₛ’s cost for all instances plus the cost of Mₗ for routed instances (Eq. 2).

The design is inspired by prior selective‑routing work (e.g., Sordoni et al., 2023; Chen et al., 2023) and emphasizes careful routing to avoid propagating flawed reasoning from smaller models.

## Experimental Setup  
- **Dataset (DQA)** – 1,879 question‑conversation pairs (≈62 % of an initial 3,000 pairs) covering 10 representative QA questions across customer experience, compliance, resolution competence, process adherence, and professionalism.  
- **Annotation** – Each pair annotated by five domain experts; only samples with ≥3 annotator agreement were kept.  
- **Prompting** – Each LM receives a question‑plan‑conversation triplet (Q, P, C). The plan P is derived from annotators’ evaluation plans. Models perform chain‑of‑thought reasoning: (1) identify evidence, (2) synthesize, (3) answer yes/no.  
- **Evaluation** – Per‑question AoI precision, recall, F1, and macro‑F1.  
- **Cost Estimation** – Token pricing from Amazon Bedrock (for *Nova*/*Claude*) and Azure OpenAI (for *OpenAI* models) projected to 1.5 M daily conversations.  
- **Baselines** – In‑house model, full inference with each large LM, and two‑tier inference with the router.  
- **Router Training** – Fine‑tuned on historical QA instances; ground truth routing labels are based on alignment between Mₛ’s prediction and gold annotation.

## Results  
| Setup | AoI Precision (%) | AoI Recall (%) | AoI F1 (%) | Macro F1 (%) | Daily Cost |
|-------|-------------------|----------------|-------------|---------------|------------|
| **Baseline Mₛ (in‑house)** | 54.2 | 64.1 | 56.1 | 67.0 | **\$3,070** |
| **Full Inference (Mₗ)** | 79.0 (nova‑pro) | 77.3 | 73.9 | 82.0 | **\$34,162** |
| | 82.3 (nova‑premier) | 78.0 | 78.8 | 85.6 | **\$114,122** |
| | 80.5 (o3‑mini) | 76.8 | 72.8 | 81.4 | **\$90,097** |
| | 79.9 (gpt‑4o) | 84.2 | 80.1 | 85.8 | **\$135,779** |
| | 72.4 (claude‑3.5‑haiku) | 83.0 | 75.5 | 82.0 | **\$43,059** |
| | **84.6 (claude‑4‑sonnet)** | 83.5 | 81.7 | 87.0 | **\$204,956** |
| **Two‑Tier Inference (Mₛ → Mₗ)** | 70.2 (nova‑pro) | 61.2 | 63.7 | 74.5 | **\$17,842** |
| | 74.3 (nova‑premier) | 71.3 | 71.3 | 79.4 | **\$60,950** |
| | 73.6 (o3‑mini) | 70.6 | 69.7 | 78.7 | **\$40,354** |
| | 73.5 (gpt‑4o) | 72.6 | 70.1 | 76.2 | **\$67,775** |
| | 65.9 (claude‑3.5‑haiku) | 61.2 | 61.3 | 72.4 | **\$20,476** |
| | **76.0 (claude‑4‑sonnet)** | 74.2 | 72.3 | 80.0 | **\$87,533** |

Key observations:

- **Cost Savings** – Using *nova‑pro* in a two‑tier setup cuts daily cost by 48 % (from \$34,162 to \$17,842).  
- **Precision Retention** – Two‑tier *nova‑pro* retains 88.9 % of full‑model AoI precision (70.2 % vs 79.0 %).  
- **Overall Trade‑off** – Across all large LMs, two‑tier inference reduces cost by 48–70 % while preserving 70–76 % of AoI precision relative to full inference.  
- **Ablation** – Experiments show that flawed reasoning from smaller models can degrade performance, underscoring careful routing design.

## Limitations  
- The paper notes that flawed reasoning from smaller models can degrade performance, emphasizing the need for careful routing design.  
- Only a single adversarial routing strategy (RoBERTa binary classifier) was evaluated; other routing functions were not explored.  
- The dataset DQA is proprietary; the authors cannot release it.  
- The study focuses on AoI precision; other business metrics (e.g., agent trust, coaching cost) are not quantified.  
- Deployment context (who deployed, scale, product) is not reported.
