# Summary

## TL;DR
The paper presents **AgentHarm‑Gen** and **Red‑Agent‑Reflect**, a fully automated red‑teaming framework that generates custom adversarial prompts and grading functions for arbitrary toolsets and iteratively refines attacks via self‑reflection. Across 115 harmful tasks (71 generated, 44 from AgentHarm) the method boosts attack success by up to **162 %** on *o4‑mini* and **86 %** on *Gemini 2.5 Pro*, demonstrating the need for agent‑specific guardrails in production systems.

## Problem & Motivation
Large Language Models (LLMs) deployed as autonomous agents with tool access introduce safety risks beyond standalone models. Existing red‑teaming tools such as **AgentHarm** rely on static prompts and hard‑coded toolsets, limiting applicability to custom production workflows that use proprietary APIs. The paper addresses this gap by automating both data generation and attack refinement for arbitrary toolsets, enabling scalable safety evaluation for real‑world agentic systems.

## Approach
The framework consists of two interlocking components:

1. **AgentHarm‑Gen**  
   * **API Definition → Tool Converter** – transforms natural‑language API specs into asynchronous Python code compatible with Inspect‑AI tools. Applied to AppWorld’s API collection, it produced **174 functional tools** (Table 5).  
   * **Adversarial Prompt Generator** – synthesizes harmful task scenarios using a template (Appendix A.2).  
   * **Automated Grading Function Generator** – creates asynchronous Python evaluation functions (Appendix A.3) that check tool usage, order, and semantic success via an LLM judge.  

2. **Red‑Agent‑Reflect**  
   * **Multi‑stage Prompting** – starts with a baseline prompt, then a detailed prompt, and iteratively refines prompts based on reflection.  
   * **Response Reflection** – after each attempt, the agent analyzes the target’s response, preserves successful tool calls, diagnoses failures, and generates a refined prompt (Algorithm 1).  
   * The process runs for a maximum of **R** iterations (default 6), selecting the best state based on a grading module.

The two components are chained: AgentHarm‑Gen produces prompts and grading functions, which Red‑Agent‑Reflect uses to generate progressively more effective attacks.

## Experimental Setup
| Item | Details |
|------|----------|
| **Dataset** | 115 harmful tasks: 71 generated by AgentHarm‑Gen, 44 from AgentHarm’s public set. 8 risk categories. |
| **Toolset** | 174 tools from AppWorld conversion. |
| **Models** | Target agents built on GPT‑4o, GPT‑4.1, *o4‑mini*, Gemini 2.5 Pro, Gemini 2.5 Flash. |
| **Red‑teaming agents** | AgentHarm‑Gen uses **Claude Sonnet 3.7**; Red‑Agent‑Reflect uses **DeepSeek R1**. |
| **Iterations** | 2 iterative refinements per prompt (total 4 attempts). |
| **Metrics** | **Refusal Rate** (percentage of prompts the agent refuses) and **Average Harm Score** (combining heuristic checks and LLM‑judge evaluation). |
| **Baseline** | AgentHarm’s original evaluation protocol. |
