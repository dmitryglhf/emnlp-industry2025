# Summary

## TL;DR
AMAS (Adaptive Multi‑Agent System) introduces a lightweight, LLM‑driven graph selector that dynamically chooses the optimal communication topology for each input sample. By replacing static, handcrafted agent graphs with a context‑aware selector trained via LoRA and REINFORCE, AMAS consistently outperforms state‑of‑the‑art single‑agent and multi‑agent baselines on question answering, mathematical deduction, and code‑generation benchmarks.

## Problem & Motivation
Large language models (LLMs) excel in many NLP tasks, yet their deployment as autonomous multi‑agent systems (MAS) is hampered by **static, handcrafted graph topologies** that ignore sample‑specific variation. Existing MAS approaches (e.g., GPTSwarm) refine a single graph via reinforcement learning, but the resulting structure remains fixed across all inputs, leading to sub‑optimal performance on diverse workloads. AMAS addresses this gap by **adapting the graph topology per query** using a lightweight LLM selector, thereby enabling context‑sensitive structural optimization.

## Approach
AMAS builds on GPTSwarm’s composite graph \(G=(N,E)\) and introduces a **dynamic graph selector**:

1. **Candidate Generation** – During training, REINFORCE optimizes a parameter vector \(\Theta\) over the composite graph (Tree‑of‑Thoughts depth 4, branching 2; Reflection agent with one reflection step over two passes; dedicated output node). Multiple checkpoints yield a pool of candidate graphs; the top‑\(K\) are retained.
2. **Selector Training** – A LoRA‑augmented LLM \(\mathcal{M}\) processes a prompt that includes the task description, the input query, and a candidate graph structure. The selector outputs a normalized score \(\hat{y}\in[0,1]\) via a pooling layer and a linear head.
3. **Ranking Loss** – For each query, the selector’s predictions are aligned with ground‑truth performance scores \(s_i\) of the \(K\) candidate graphs. The loss
   \[
   \mathcal{L}_r=\sum_{i\neq j} m(i,j)\cdot g(i,j)
   \]
   where \(m(i,j)=\max(0,|r_j-r_i|^{0.5})\) and \(g(i,j)=\text{GeLU}((s_j-s_i)(\hat{y}_j-\hat{y}_i))\), enforces list‑wise ranking semantics.
4. **Inference** – For a new query, the selector evaluates all \(K\) candidate graphs and selects the one with the highest predicted score, thereby routing the query through the most suitable agent topology.

- **Baselines**:  
  - *Monolithic agents*: Input‑Output (IO), Chain‑of‑Thought (COT), Self‑Consistency, Tree‑of‑Thought (TOT), Graph‑of‑Thought (GOT).  
  - *Collaborative ecosystems*: AutoGPT, AgentVerse, GPTSwarm.  
- **LLMs**: GPT‑3.5‑turbo, LLaMA‑3 (8B & 70B), distilled Deepseek R1 (7B).  
- **Hardware**: NVIDIA A40 (48 GB) or A100 (80 GB).  
- **Graph Configuration**: Composite graph with 12 nodes (ToT depth 4, branching 2; Reflection agent; output node).  
- **Training**: LoRA adaptation (~0.5 % of backbone memory), REINFORCE for graph optimization.

## Limitations
- **Reinforcement learning instability**: Automated topological optimization remains challenging due to RL’s inherent instability.  
- **Static graph limitation in prior work**: Existing MAS approaches enforce a single graph (handcrafted or RL‑optimized), neglecting sample‑specific heterogeneity. AMAS mitigates this but still relies on RL for candidate generation, inheriting its instability.  
- **No deployment or A/B‑test results** are provided, limiting insight into real‑world performance gains.
