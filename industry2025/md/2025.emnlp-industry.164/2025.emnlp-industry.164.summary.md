# Summary

## TL;DR
Recover‑LoRA is a lightweight, data‑agnostic method that restores accuracy lost in small language models (SLMs) due to silent weight corruption (e.g., improper serialization). By training low‑rank LoRA adapters on selective layers with logit distillation from synthetic data, it recovers 5–17 % of the lost accuracy on MHA and GQA‑style SLMs, outperforming both LLM‑QAT* and dataset‑specific LoRA fine‑tuning on most tested models.

## Problem & Motivation
Inference optimizations such as quantization, pruning, datatype conversion, and model export can silently corrupt model weights, leading to significant downstream task performance drops. Existing solutions focus mainly on robust quantization; they do not address other sources of degradation like improper serialization or framework conversion. In edge deployments, where compute and memory budgets are tight, such silent failures can render a model unusable. Recover‑LoRA targets this real‑world problem by providing a method that does not require labeled data or full‑model retraining, making it suitable for industry scenarios where proprietary data is scarce and retraining large models is costly.

## Approach
Recover‑LoRA learns a small set of LoRA adapters \(A\) and \(B\) that modify the frozen, corrupted weight matrix \(W_s\) of the degraded model \(M_S\).  
1. **Synthetic data** \(D_{\text{syn}}\) is generated via the hybrid sampling strategy from LLM‑QAT (deterministic first 3–5 tokens, stochastic remainder).  
2. **Logit distillation** aligns the student \(M_S\) with the full‑precision teacher \(M_T\) by minimizing the KL divergence between their softmax logits.  
3. **Selective layer training**: adapters are added only to key layers (e.g., K, V projections for AMD‑OLMo‑SFT 1B; ATTN, MLP for Llama3.2 1B; ATTN, MLP for Gemma2 2B; K, V for DeepSeek‑R1‑Distill‑Qwen 1.5B).  
4. The training objective is  
   \[
   L_{\text{KD}} = \sum_i p_t^i \log\frac{p_t^i}{p_s^i}
   \]
   where \(p_t\) and \(p_s\) are teacher and student softmax probabilities.

This design keeps the method lightweight (few parameters) and dataset‑agnostic.

## Experimental Setup
| Component | Details |
|-----------|---------|
| **Models** | AMD‑OLMo‑SFT 1B, Llama3.2 1B, Gemma2 2B, DeepSeek‑R1‑Distill‑Qwen 1.5B |
| **Baselines** | LLM‑QAT* (logit distillation on all parameters, no quantization simulation), SFT‑LoRA (cross‑entropy on labeled data) |
| **Synthetic data** | Generated with hybrid sampling (Appendix A) |
| **Evaluation datasets** | PiQA, OpenBookQA, WinoGrande, BoolQ, HellaSwag, Arc C, MMLU (Philosophy, Management, Astronomy) |
| **Metrics** | Accuracy Recovery Percentage (AR%) defined as \(\frac{(E_S^* - E_S)}{|E_S - E_T|}\times100\) |
| **Hardware** | AMD MI300X GPUs |
| **Hyperparameters** | Full details in Appendix B |

Table 1 reports L2‑norm differences between original and perturbed weights for the first K‑projection layer: AMD‑OLMo‑SFT 1B = 44.06, Llama3.2 1B = 52.97, Gemma2 2B = 35.94, DeepSeek‑R1‑Distill‑Qwen 1.5B = 40.69.

Key observations:
- Recover‑LoRA consistently outperforms LLM‑QAT* and SFT‑LoRA on 3 of the 4 models.  
- Average accuracy recovery ranges from 5 % to 17 % on MHA and GQA SLMs.  
- Figure 2 shows that Recover‑LoRA trains far fewer parameters and uses less data than the baselines, highlighting its efficiency.
