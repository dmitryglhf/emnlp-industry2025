# Summary

## TL;DR  
The paper presents a **Multi‑Agent framework** that augments a ReAct‑based Base‑Agent with reflection, domain‑specific insights, and human‑in‑the‑loop tools to tackle portfolio‑management analytics. On a benchmark of 100 synthetic finance questions, the Multi‑Agent achieves a **Pass@1 of 46.00 %**—the highest among four evaluated setups—demonstrating its relevance for industry‑grade quantitative finance tasks.

## Problem & Motivation  
Portfolio managers routinely perform complex quantitative tasks (statistical analysis, portfolio modeling, back‑testing, scenario analysis). Existing LLMs (e.g., GPT‑4o) struggle with:

- **Hallucinations** and subtle bugs in generated code.  
- **Incomplete workflows** (e.g., missing missing‑value handling).  
- **Wrong column or formula usage** (e.g., mis‑calculating P/E ratios).  
- **Misinterpretation of financial statements** (balance sheets, cash‑flow).  

These shortcomings limit LLM deployment in real‑world finance. The authors therefore created a **benchmark of 100 domain‑specific questions** (63 hard, 37 easy; 62 finance, 38 statistics) to quantify LLM performance and motivate a more robust, multi‑agent solution.

## Approach  
The framework is a **pipeline of LLM‑powered agents**:

| Agent | Role |
|-------|------|
| **Base‑Agent** | ReAct agent that generates Python code, executes it, and returns results. |
| **Reflection Agent** | Iteratively critiques and corrects the Base‑Agent’s outputs. |
| **Insights Agents** (Data Summarizer, Question Generator, Finance Expert, Query Refiner) | Provide domain knowledge, refine user queries, and summarize data. |
| **Ask Human Tool** | Allows human oversight when the agent encounters errors or ambiguous inputs. |

The Base‑Agent uses a **Python code execution tool**; errors trigger retries or human intervention. The Insights agents pre‑process the data file and user query, ensuring the Base‑Agent receives a clear, domain‑aware prompt. Reflection adds a second layer of scrutiny, improving precision and completeness.

## Experimental Setup  
- **Dataset**: 100 synthetic finance questions (generated manually with portfolio‑manager input).  
- **Question categories**: 63 hard, 37 easy; 62 finance, 38 statistics.  
- **Baselines**:  
  1. Base‑Agent (single ReAct agent).  
  2. Base‑Agent + Reflection.  
  3. Base‑Agent + Insights.  
  4. **Multi‑Agent** (full pipeline).  
- **LLM**: OpenAI’s GPT‑4o (same model across all setups).  
- **Evaluation**: Each question answered 10 times; correctness measured by exact match to predetermined numeric/string answers.  
- **Metrics**: Pass@k for k ∈ {1, 2, 5, 10}.  
- **Tool**: Python code execution; intermediate code not evaluated.

## Results  
| Framework | Pass@1 | Pass@2 | Pass@5 | Pass@10 |
|-----------|--------|--------|--------|---------|
| Base‑Agent | 39.59 | 45.55 | 51.85 | 55.0 |
| Base‑Agent + Reflection | 39.00 | 46.62 | 53.13 | 56.0 |
| Base‑Agent + Insights | 43.89 | 53.84 | 64.87 | 72.0 |
| **Multi‑Agent** | **46.00** | 57.16 | 67.69 | 72.0 |

The Multi‑Agent outperforms all baselines, achieving the highest Pass@1 (46.00 %) and matching the best Pass@10 (72.0 %) of the other setups.
