# Summary

## TL;DR  
HOLA is an end‑to‑end framework that accelerates LLM inference on edge devices by combining **Hierarchical Speculative Decoding (HSD)**, **AdaComp‑RAG** (adaptive retrieval), and **Lo‑Bi** (structured pruning + mixed‑precision quantization).  On GSM8K and ARC, HOLA boosts Exact‑Match Accuracy (EMA) by up to **+17.6 %** and Multiple‑Choice Accuracy (MCA) by **+10.5 %**, while cutting latency by ~50 ms and memory by ~750 MB on Jetson Nano‑style hardware.

## Problem & Motivation  
Large Language Models (LLMs) are compute‑ and memory‑hungry, limiting their deployment in real‑time edge scenarios such as healthcare diagnostics, educational tutoring, and embedded assistants.  Existing optimizations—quantization, pruning, or Retrieval‑Augmented Generation (RAG)—only partially relieve these constraints and often trade off speed, accuracy, or generality.  The paper targets the full inference pipeline, aiming to make LLMs practical for low‑resource, production‑ready use cases.

## Approach  
HOLA unifies three complementary modules:

| Module | Core Idea | Key Design Choices |
|--------|-----------|--------------------|
| **HSD** | Draft‑and‑verify generation with a lightweight draft model and a higher‑fidelity verifier, gated by token‑entropy (`τ = 1.5`). | Reduces verifier calls from **O(T)** to **O(k)**, achieving 2–3× speedup without quality loss. |
| **AdaComp‑RAG** | Adaptive retrieval depth based on query uncertainty (`C(q) = ‖∇_q 𝓛‖₂`, threshold `δ = 0.85`). | Retrieves `k = 3` documents via a BERT‑based dense encoder; compositional attention merges retrieved and original keys. |
| **Lo‑Bi** | Structured pruning (LoRA, rank `r = 4`) followed by mixed‑precision quantization (4/8/16‑bit). | Subblocks are assigned precision by minimizing expected quantization error; ~70 % of blocks use 8‑bit, critical paths keep 16‑bit. |

## Experimental Setup  
| Component | Details |
|-----------|----------|
| **Models** | GPT‑2, TinyLlama, LLaMA‑3.2‑3B, Phi‑1.5, Phi‑3.5‑mini, Gemma‑2B, Gemma‑7B, Mistral‑3B, Mistral‑7B |
| **Datasets** | **GSM8K** (≈ 8 k arithmetic/algebra problems) and **ARC** (AI2 Reasoning Challenge, multiple‑choice science questions) |
| **Metrics** | EMA (Exact‑Match Accuracy) on GSM8K; MCA (Multiple‑Choice Accuracy) on ARC; latency (ms) and memory (MB) |
| **Hardware** | NVIDIA A100 GPUs (80 GB) for training; Jetson Nano‑style edge devices for inference |
| **Hyper‑parameters** | `τ = 1.5`, `δ = 0.85`, `r = 4`, AdamW (`lr = 2×10⁻⁴`), batch 32, linear warm‑up 10 %, early stopping on EMA/MCA |
| **Code** | https://github.com/zohaibhasan066/HOLA_Codebase |
