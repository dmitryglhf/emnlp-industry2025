# Summary

## TL;DR  
The paper presents a multi‑task pre‑finetuning framework for lightweight transformer encoders that uses task‑primary LoRA modules to reconcile the conflicting objectives of NER and text classification. The method yields a single shared backbone with modular adapters, achieving +0.8 % F1 on NER and +8.8 % accuracy on 16 text‑classification datasets, making it well‑suited for on‑device mobile NLP.

## Problem & Motivation  
Mobile NLP applications (e.g., automatic calendar event creation, personalized recommendations) require models that can be reused across many tasks while staying small in memory and computation. Existing solutions either fine‑tune a pre‑trained BERT‑like encoder—often sub‑optimal when downstream data is limited—or deploy separate encoders per task, which is infeasible on-device. The authors argue that a pre‑finetuning stage, performed on large‑scale data with objectives closer to the target tasks, can bridge this gap, but naïve multi‑task pre‑finetuning degrades performance due to conflicting optimisation signals.

## Approach  
1. **Individual pre‑finetuning**  
   * **NER** – Knowledge distillation from a NuNER model trained on a 24.4 M‑word dataset extracted by ChatGPT‑3.5. The lightweight encoder (MiniLM or DistilBERT) is trained to predict pseudo‑labels derived from NuNER’s token embeddings.  
   * **Text Classification** – Weakly supervised contrastive learning on ~895 M text pairs compiled from SentenceTransformers. The InfoNCE loss pulls semantically similar pairs together while pushing others apart.  

2. **Multi‑task pre‑finetuning with Task‑Primary LoRAs (MTPF‑TPL)**  
   * Extend the encoder with two groups of LoRA modules applied to the last few transformer layers.  
   * Each LoRA is updated only with its associated task loss, while the backbone is jointly updated by both losses.  
   * After pre‑finetuning, the backbone is deployed as the shared model; the LoRA modules are distributed to applications as adapters.  

Key design choices: using LoRA instead of freezing the backbone, allowing joint optimisation, and keeping the number of parameters proportional to the number of downstream tasks.

## Experimental Setup  
| Component | Details |
|-----------|----------|
| **Datasets** | **NER**: CoNLL (4 types), MIT Restaurant (8), BioNLP (5), OntoNotes5 (18), CrossNER (39). <br> **Text Classification**: Banking77, Emotion, AmazonCounterfactual, MassiveIntent, TweetSentiment, ToxicChat, News, Patent, FinancialPhrasebank, FrenkEn, IMDB, ArXiv, DBpedia, TweetTopicSingle, YelpReviewFull, ToxicConversation (16 total). |
| **Models** | MiniLM (33 M params) and DistilBERT (67 M params), English‑only versions. |
| **Baselines** | Base pre‑trained model (no pre‑finetuning). |
| **Evaluation metrics** | F1 for NER, accuracy for TC. |
| **Adaptation** | TC: linear classifier on mean‑pooled token embeddings; NER: token‑classification with LoRA. |
| **Training details** | Contrastive learning: InfoNCE loss with τ = 0.05, in‑batch negatives. Knowledge distillation: pseudo‑labels from NuNER token embeddings via mini‑batch k‑means. |

## Results  
| Model | Approach | NER (5 DS) | TC (16 DS) | Avg |
|-------|----------|-------------|-------------|-----|
| Base | – | 76.4 | 55.1 | 65.8 |
| PF for NER | – | 77.1 (+0.7) | 53.9 (‑1.2) | 65.5 (‑0.3) |
| PF for TC | – | 74.6 (‑1.8) | 63.9 (+8.8) | 69.2 (+3.4) |
| MTPF‑TPL | – | 77.1 (+0.8) | 63.9 (+8.8) | 70.5 (+4.7) |

- Individual pre‑finetuning improves its own task family: +0.7 % F1 for NER, +8.8 % accuracy for TC.  
- Naïve multi‑task pre‑finetuning degrades performance.  
- MTPF‑TPL achieves performance comparable to the best individual pre‑finetuning while providing a single shared backbone and task‑specific LoRA adapters, satisfying deployment constraints.  
- Across 21 downstream tasks, the method yields average gains of +0.8 % for NER and +8.8 % for text classification.
