# Summary

## TL;DR  
Ryt AI is the first globally regulator‑approved, LLM‑native conversational banking system that lets customers execute core financial transactions—such as fund transfers and bill payments—through natural language. Built on an in‑house 10 B‑parameter *ILMU* model with LoRA adapters, the system uses deterministic guardrails, human‑in‑the‑loop confirmation, and a stateless audit architecture to meet strict compliance and security requirements.

## Problem & Motivation  
Legacy digital banking workflows are multi‑screen, rigid, and offer little semantic understanding, causing friction and limited usability. Existing LLM assistants are confined to advisory or support roles and cannot safely perform high‑stakes operations. Ryt Bank, the world’s first AI‑native licensed bank, needed a solution that replaces the 30–45 s multi‑screen transfer flow with a single conversational interface while satisfying regulatory and security constraints.

## Approach  
- **LLM‑native agentic framework**: Four specialized agents—Guardrails, Intent, Payment, FAQ—each attaches a task‑specific LoRA adapter to the *ILMU* backbone.  
- **Guardrails**: Instruction‑tuned *ILMU* classifies inputs into violation categories (e.g., code‑interpreter abuse, privacy breach) and blocks malicious prompts.  
- **Intent Classifier**: Semantic reasoning over the current message and conversation history to route requests to the appropriate downstream agent. Fine‑tuned on ~10⁵ anonymized instruction examples reflecting noisy Malaysian banking language.  
- **Payment Agent**: Extracts five core fields (recipient name, bank name, account/ID, amount, reference), validates via a schema validator, supports image‑based inputs through integrated OCR, and submits to the backend gateway for balance checks, transaction limits, and AML screening. Human‑in‑the‑loop confirmation is required; 2FA is triggered when P2P transfers or cumulative daily amounts exceed RM250, and for P2M payments when the amount exceeds RM250.  
- **FAQ Agent**: Retrieval‑Augmented Generation (RAG) pipeline (Lewis et al., 2021) reformulates ambiguous queries, retrieves from a vector store of banking FAQs, reranks, and generates factually grounded responses.  
- **Implementation**: *ILMU* is a decoder‑based model with 8 K token context window and rotary positional embeddings (RoPE). The system is stateless, session‑bound, and passes a shared JSON context between agents to preserve traceability.
