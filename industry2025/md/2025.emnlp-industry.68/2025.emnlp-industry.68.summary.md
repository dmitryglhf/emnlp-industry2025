# Summary

## TL;DR
UniEDU is a unified large multimodal model that handles four core K‑12 educational tasks—knowledge recommendation, knowledge tracing, time‑cost prediction, and user answer prediction—within a single generative framework. By compressing up to 300 interaction histories into a compact token set and leveraging Flash Attention, the model achieves roughly a 3× reduction in VRAM usage compared to state‑of‑the‑art LMMs while delivering competitive or superior performance (≈30 % gain on knowledge recommendation, ≈20 % on time‑cost prediction) over task‑specific baselines.

## Problem & Motivation
K‑12 learning materials are inherently multimodal (text + images), yet most AI systems focus on plain text and are task‑specific. Existing solutions (e.g., SASRec, Bert4Rec for recommendation; extraKT, reKT for tracing) cannot share knowledge across tasks and suffer from high computational overhead when handling long interaction histories (up to 45 000 tokens). The industry context demands a single model that can generalize across tasks, handle multimodal inputs, and be efficient enough for real‑world deployment.

## Approach
- **Unified Generative Framework**: All tasks are reframed as sequence generation conditioned on a user’s interaction history and a task instruction.
- **Profile Encoder**: Processes the raw interaction sequence \(S=\{X_1,\dots,X_n\}\), where each \(X_i=\{q_i,a_i,k_i,c_i,t_i\}\) (question stem, answer, knowledge concept, correctness, time). The encoder compresses each interaction into \(m\) tokens, producing a matrix \(|S|\times m\times h\).
- **Projector**: A linear layer maps encoder outputs to the hidden dimension \(h\) expected by the language model.
- **Language Model**: A transformer (Qwen2.5‑0.5B/1.5B/3B) that receives the compressed profile embeddings plus the uncompressed task instruction embeddings and autoregressively generates the task‑specific output.
- **Training Objective**: Standard autoregressive loss \(p(\mathbf{X}_t|S,\mathbf{X}_{inst})=\prod_i p_\theta(x_i|S,\mathbf{X}_{inst},\mathbf{X}_{t,<i})\).
- **Efficiency Optimizations**: Flash Attention reduces attention memory from quadratic to linear in sequence length; VRAM analysis shows UniEDU uses 38.8 GB (training) and 9.5 GB (inference) for a 5B‑parameter model versus 127.4 GB/31.5 GB for Qwen2‑VL‑2B.

## Experimental Setup
- **Dataset**: Real student exercise data from a popular e‑learning platform.  
  - 13,239 students, 8,247 knowledge concepts, 235,687 questions, 3,892,084 interactions.  
  - Each student’s history truncated to ≤300 interactions (token length ≤45 000).  
- **Tasks**:  
  1. *Knowledge Recommendation* – rank \(K\in\{5,10,25,50\}\) candidate concepts; evaluated with Precision@1.  
  2. *Knowledge Tracing* – binary prediction of correct answer; evaluated with accuracy.  
  3. *Time‑Cost Prediction* – integer time prediction; evaluated with MAE.  
  4. *User Answer Prediction* – generate the most probable answer; evaluated with Exact Match (EM).  
- **Baselines**:  
  - Recommendation: SASRec, Bert4Rec.  
  - Tracing: extraKT, reKT.  
  - Time‑cost: N‑BEATS, Prophet.  
  - Answer: Qwen2‑VL‑2B, Qwen2‑VL‑7B (both un‑fine‑tuned).  
- **Training Regimen**: 24,504 sequences for training, 5 % validation, 2,784 test.  
- **Model Configurations**: Encoder fixed to Qwen2‑VL‑2B; language model sizes Qwen2.5‑0.5B, 1.5B, 3B → UniEDU‑2.5B, 3.5B, 5B.

## Results
- **Overall Performance**: UniEDU outperforms task‑specific baselines on Knowledge Recommendation, Time‑Cost Prediction, and User Answer Prediction.  
- **Quantitative Gains**:  
  - Knowledge Recommendation: ≈30 % improvement over best baseline (SASRec/Bert4Rec).  
  - Time‑Cost Prediction: ≈20 % improvement over best baseline (N‑BEATS/Prophet).  
  - Knowledge Tracing: Comparable to extraKT/reKT; slightly lower accuracy.  
- **Efficiency**: VRAM usage reduced by ≈3× relative to Qwen2‑VL‑2B (training: 38.8 GB vs 127.4 GB; inference: 9.5 GB vs 31.5 GB).  
- **Model Size Impact**: Larger language models (UniEDU‑5B) yield marginal gains over smaller variants but maintain the same efficiency advantage.
