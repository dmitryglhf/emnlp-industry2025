# Summary

## TL;DR  
The paper introduces **PI‑LoRA (Path‑Integrated LoRA)**, a new low‑rank adaptation technique that improves rank allocation for LoRA modules by integrating gradient‑path information and Shapley‑value inspired importance scoring.  On the **Text2MDT** benchmark, PI‑LoRA outperforms all existing parameter‑efficient fine‑tuning (PEFT) baselines (LoRA, AdaLoRA, AutoLoRA, MOELoRA, DoRA, Parallel‑Adapter, Learned‑Adapter, P‑tuning v2, IAPT, BitFit, SSP) while keeping the number of tunable parameters comparable or lower.  The method is especially relevant to industry because it delivers state‑of‑the‑art accuracy with a lightweight model that is suitable for clinical decision‑support systems.

## Problem & Motivation  
Medical decision processes are best represented as **medical decision trees (MDTs)**.  Extracting MDTs from clinical guidelines and textbooks would drastically reduce the need for labor‑intensive manual annotation and enable robust decision‑support tools.  Existing solutions rely on large‑scale manual effort and suffer from sub‑optimal rank allocation in LoRA‑based PEFT methods:  
- **AdaLoRA** uses sensitivity‑based scores that ignore interactions between parameters.  
- **SoRA** and **SaLoRA** rely on architectural parameters that do not reliably reflect rank importance.  
These shortcomings motivate a new rank‑allocation strategy that captures synergistic effects between modules.

## Approach  
**PI‑LoRA** treats each LoRA module as a participant in a cooperative game and scores its importance by integrating gradients along a path from a zero baseline to the current parameter value.  
1. **Importance scoring**  
   \[
   s(w)=\left|w\int_{0}^{1}\nabla_{w}\mathcal{L}(\alpha w)d\alpha\right|
   \approx\frac{|w|}{K}\Big\|\sum_{k=1}^{K}\nabla\mathcal{L}\!\big(\tfrac{k}{K}w\big)\Big\|
   \]  
   (Eq. 5–6).  
2. **Stochastic approximation** – during each training step \(p\) a random \(\alpha_{p}\) is sampled and a single‑point estimate \(\tilde{s}^{(p)}(w)\) is computed (Eq. 7).  
3. **Temporal aggregation** – after \(P\) mini‑batches the final score is \(\tilde{s}(w)=\frac{1}{P}\sum_{p}\tilde{s}^{(p)}(w)\) (Eq. 8).  
4. **Module‑level score** – the average over all parameters in a LoRA module gives \(\tilde{s}(\text{LoRA}_{m,l})\) (Eq. 9).  
5. **Pruning** – modules with the lowest scores are removed until the target number of LoRA modules \(N_{\text{LoRA}}\) is reached (Algorithm 1).  

Implementation details:  
- Each LoRA module is initialized with rank \(r=16\).  
- \(K=25\) integration points are used.  
- At the end of training half of the LoRA modules are pruned; the remaining modules are re‑initialized and retrained.  
- The backbone is **Qwen 2.5 7B** (open‑source LLM).  Ablations also use **Baichuan 2 7B** and **GLM‑4‑9B‑Chat**.  
- Training uses Adam (learning rate \(1\times10^{-4}\), warm‑up 100 steps, batch size 32 with gradient accumulation).  
- Loss is MSE; beam search with beam size 3 is used at inference.  
- Validation is performed after each epoch; training stops if validation loss does not improve for 5 epochs.  
- In the final fine‑tuning stage, all LoRA modules are re‑initialized according to the pruned allocation and trained with the same hyper‑parameters; evaluation on the dev set occurs every 200 steps with patience 10.

## Experimental Setup  
- **Task**: Text2MDT – reconstruct medical decision trees from clinical texts.  
- **Evaluation metrics**  
  - Triplet extraction: precision, recall, F1 (as in Zhu et al., 2023).  
  - Node grouping: Levenshtein‑ratio style score **NG LR**.  
  - Tree assembling / overall Text2MDT:  
    - Tree accuracy (**Tree Acc**)  
    - Decision‑path F1 (**DP_F1**)  
    - Tree Levenshtein ratio (**Tree_LR**).  
- **Baselines**:  
  - LoRA variants: LoRA, AdaLoRA, AutoLoRA, MOELoRA, DoRA.  
  - Other PEFT methods: Parallel‑Adapter, Learned‑Adapter, P‑tuning v2, IAPT, BitFit, SSP.  
- **Deployment context**: offline fine‑tuning (supervised fine‑tuning, SFT).  No extra prediction heads; predictions are generated via the language‑model head.  Beam search (size 3) is used during inference.

## Results  
- **Table 1** (not shown) reports that PI‑LoRA outperforms all baseline PEFT methods across **all seven sub‑tasks** (triplet extraction, node grouping, tree assembling) while keeping tunable parameters comparable.  
- PI‑LoRA surpasses the previous SOTA LoRA‑based baselines (AdaLoRA, AutoLoRA, DoRA, MOELoRA) with comparable or fewer tunable parameters.  
- **Table 2** (not shown) confirms that PI‑LoRA achieves superior performance in the end‑to‑end framework, outperforming all other methods.  
- Overall, PI‑LoRA delivers state‑of‑the‑art accuracy on the Text2MDT benchmark with a lightweight architecture.
